=== [0] ===
PROMPT: What is the difference between an abstract class and an interface in Java?
ANSWER: An abstract class can have state (fields), constructors, and concrete methods. An interface (Java 8+) can have default and static methods but no instance state. A class can implement multiple interfaces but extend only one abstract class. Use abstract classes for shared state/behavior, interfaces for capability contracts.

=== [1] ===
PROMPT: Explain the SOLID principles in Java.
ANSWER: S - Single Responsibility: a class has one reason to change.
O - Open/Closed: open for extension, closed for modification.
L - Liskov Substitution: subtypes must be substitutable for their base types.
I - Interface Segregation: prefer many small interfaces over one large one.
D - Dependency Inversion: depend on abstractions, not concretions.

=== [2] ===
PROMPT: What is the difference between method overloading and method overriding?
ANSWER: Overloading: same method name, different parameter lists, resolved at compile time (static dispatch). Overriding: subclass provides a new implementation for a superclass method with the same signature, resolved at runtime (dynamic dispatch). @Override annotation helps catch mistakes at compile time.

=== [3] ===
PROMPT: What is the difference between composition and inheritance?
ANSWER: Inheritance (IS-A): subclass inherits all behaviour; tight coupling, fragile base class problem. Composition (HAS-A): class contains a reference to another object; loose coupling, easier to change. Prefer composition over inheritance unless a true IS-A relationship exists.

=== [4] ===
PROMPT: What are access modifiers in Java and what do they control?
ANSWER: private – only within the declaring class.
package-private (no modifier) – within the same package.
protected – same package + subclasses.
public – everywhere.
They control encapsulation and API visibility.

=== [5] ===
PROMPT: What is the 'final' keyword in Java and where can it be applied?
ANSWER: final class – cannot be subclassed (e.g. String).
final method – cannot be overridden.
final variable/field – can be assigned only once; if primitive it is a constant, if reference the reference cannot change but the object may be mutated.

=== [6] ===
PROMPT: What is the difference between 'static' and instance members?
ANSWER: Static members belong to the class and are shared across all instances; they exist without any object. Instance members belong to each object. Static methods cannot access instance fields or call instance methods without a reference.

=== [7] ===
PROMPT: Explain the concept of polymorphism with an example.
ANSWER: Polymorphism means 'many forms'. In Java it is achieved via method overriding and interfaces. Example: Animal ref = new Dog(); ref.speak(); calls Dog.speak() at runtime even though the compile-time type is Animal. This enables writing generic algorithms that work with any subtype.

=== [8] ===
PROMPT: What is encapsulation and why is it important?
ANSWER: Encapsulation hides internal state behind a public API (getters/setters). Benefits: prevents invalid state, allows changing implementation without breaking callers, makes classes easier to reason about and test.

=== [9] ===
PROMPT: What is a constructor, and what happens if you don't define one?
ANSWER: A constructor initialises an object. If no constructor is defined, the compiler provides a default no-arg constructor. If you define any constructor the default is no longer provided automatically.

=== [10] ===
PROMPT: What is a marker interface and can you give an example?
ANSWER: A marker interface has no methods; it conveys metadata to the JVM or frameworks. Examples: java.io.Serializable, java.lang.Cloneable. Modern Java often uses annotations instead (e.g. @FunctionalInterface).

=== [11] ===
PROMPT: What is the difference between == and equals() in Java?
ANSWER: == compares reference identity (or primitive values). equals() compares logical equality as defined by the class. For String, Integer, etc. equals() compares content. Always override hashCode() when overriding equals().

=== [12] ===
PROMPT: What is an inner class and what are the types in Java?
ANSWER: 1. Static nested class – no reference to outer instance.
2. Inner (non-static) class – holds implicit reference to outer instance.
3. Local class – declared inside a method.
4. Anonymous class – unnamed, declared and instantiated in one expression.
Lambdas replace many anonymous class uses since Java 8.

=== [13] ===
PROMPT: What are default methods in interfaces (Java 8+)?
ANSWER: Default methods provide a method implementation in an interface, allowing new methods to be added without breaking existing implementations. They are declared with the 'default' keyword. If two interfaces provide the same default method, the implementing class must override it to resolve the conflict.

=== [14] ===
PROMPT: What is the difference between checked and unchecked exceptions?
ANSWER: Checked exceptions (extend Exception) must be declared or caught; they represent recoverable conditions (IOException, SQLException). Unchecked exceptions (extend RuntimeException) do not need to be declared; they represent programming errors (NullPointerException, ArrayIndexOutOfBoundsException).

=== [15] ===
PROMPT: What is covariant return type in Java?
ANSWER: Since Java 5, an overriding method may return a subtype of the return type declared in the superclass. This avoids casting when working with the concrete subtype. Example: a clone() override can return the specific class type instead of Object.

=== [16] ===
PROMPT: Explain the Liskov Substitution Principle with a concrete counter-example.
ANSWER: LSP states that objects of a supertype should be replaceable by objects of a subtype without altering correctness. Classic violation: Square extends Rectangle and overrides setWidth/setHeight to keep sides equal – code expecting a Rectangle and setting different width and height breaks. Better to use a common Shape abstraction.

=== [17] ===
PROMPT: What is the difference between an abstract class with no abstract methods and an interface?
ANSWER: An abstract class with no abstract methods can have constructors, instance fields, and full method implementations – it mainly prevents direct instantiation. An interface cannot have instance fields or constructors. The distinction matters when designing APIs where you want to force subclassing vs. implementing a contract.

=== [18] ===
PROMPT: What is a functional interface in Java 8?
ANSWER: An interface with exactly one abstract method (SAM). It may have default and static methods. The @FunctionalInterface annotation is optional but recommended. Examples: Runnable, Callable, Comparator, Predicate<T>, Function<T,R>. Lambdas and method references can be used wherever a functional interface is expected.

=== [19] ===
PROMPT: Explain the difference between shallow copy and deep copy in Java.
ANSWER: Shallow copy copies the object and copies references to contained objects (both copies point to same nested objects). Deep copy recursively copies nested objects so both copies are fully independent. Object.clone() performs a shallow copy by default. Deep copy requires custom logic or serialisation.

=== [20] ===
PROMPT: What is method hiding vs method overriding?
ANSWER: Overriding applies to instance methods; the version called is determined at runtime by the actual object type. Hiding applies to static methods; the version called is determined at compile time by the reference type. Static methods cannot be overridden, only hidden.

=== [21] ===
PROMPT: What is the difference between Comparable and Comparator?
ANSWER: Comparable defines the natural ordering via compareTo() and is implemented by the class itself. Comparator defines an external ordering strategy and can be passed to sort methods. Use Comparable for natural order, Comparator for multiple or external orderings. Comparator.comparing() factory methods make it concise.

=== [22] ===
PROMPT: What is object cloning and what are its pitfalls?
ANSWER: Cloning creates a copy via Object.clone(), which requires implementing Cloneable. Pitfalls: default shallow copy may share mutable state; Cloneable is a marker interface but clone() is defined in Object; checked CloneNotSupportedException; copy constructors or static factory methods are often preferred alternatives.

=== [23] ===
PROMPT: What is the builder pattern and when should you use it?
ANSWER: Builder separates construction of a complex object from its representation. Use when an object has many optional parameters or when immutability is desired. A nested static Builder class accumulates parameters and a build() method returns the immutable product. Lombok's @Builder automates this.

=== [24] ===
PROMPT: What is the singleton pattern and how can you implement it thread-safely?
ANSWER: Singleton ensures only one instance exists. Thread-safe implementations:
1. Eager init: private static final instance.
2. Synchronized getInstance().
3. Double-checked locking with volatile.
4. Enum singleton (recommended): enum MyEnum { INSTANCE; } – thread-safe, serialization-safe.

=== [25] ===
PROMPT: What is the difference between ArrayList and LinkedList?
ANSWER: ArrayList uses a dynamic array; O(1) random access, O(n) insert/delete in middle, good cache locality. LinkedList uses a doubly-linked list; O(n) access, O(1) insert/delete at known node, more memory (node overhead). ArrayList is preferred for most use cases.

=== [26] ===
PROMPT: What is the contract between equals() and hashCode()?
ANSWER: If two objects are equal (equals() returns true), they must have the same hashCode(). The reverse is not required (hash collisions are allowed). Violation breaks HashMap/HashSet behaviour: objects may be lost or duplicated in hash-based collections.

=== [27] ===
PROMPT: How does HashMap work internally in Java?
ANSWER: HashMap uses an array of buckets. The key's hashCode() is hashed to compute the bucket index. Collisions are handled via linked lists (chains); since Java 8 chains longer than 8 entries become TreeNodes (red-black tree) for O(log n) worst case. Default load factor 0.75 triggers resizing when 75% full.

=== [28] ===
PROMPT: What is the difference between HashMap, LinkedHashMap, and TreeMap?
ANSWER: HashMap – no ordering, O(1) average get/put. LinkedHashMap – insertion (or access) order maintained via doubly-linked list, slightly slower. TreeMap – sorted by natural/comparator order, O(log n) operations, implements NavigableMap. Choose based on ordering requirements.

=== [29] ===
PROMPT: What is the difference between HashSet and TreeSet?
ANSWER: HashSet – backed by HashMap, O(1) average, no order. TreeSet – backed by TreeMap, O(log n), sorted natural/comparator order, implements NavigableSet. LinkedHashSet provides insertion-order iteration.

=== [30] ===
PROMPT: What are Java Streams and what are their key characteristics?
ANSWER: Streams (java.util.stream) provide a pipeline of operations on sequences. Key traits: lazy evaluation (intermediate ops defer until terminal op), internal iteration (no explicit loops), potentially parallelisable, do not mutate source, one-shot (cannot be reused after terminal op).

=== [31] ===
PROMPT: What is the difference between map() and flatMap() in Streams?
ANSWER: map() transforms each element 1-to-1 (returns Stream<Stream<T>> if mapper returns Stream). flatMap() transforms each element to a stream and flattens all streams into one – useful when each element maps to multiple results (e.g. splitting sentences into words).

=== [32] ===
PROMPT: What is the difference between intermediate and terminal Stream operations?
ANSWER: Intermediate operations (filter, map, sorted, distinct, limit) are lazy – they return a new Stream and are not executed until a terminal operation is called. Terminal operations (collect, forEach, reduce, count, findFirst) trigger the pipeline and produce a result or side effect.

=== [33] ===
PROMPT: Explain the Collectors.groupingBy() collector.
ANSWER: groupingBy classifies stream elements into groups via a classifier function and returns a Map<K, List<T>>. You can supply a downstream collector: groupingBy(classifier, counting()) to get Map<K,Long>, or groupingBy(classifier, Collectors.toSet()). Useful for categorisation similar to SQL GROUP BY.

=== [34] ===
PROMPT: What is Optional in Java and why was it introduced?
ANSWER: Optional<T> is a container that may or may not hold a non-null value. It forces callers to handle the absent case, reducing NullPointerExceptions. Key methods: isPresent(), get(), orElse(), orElseGet(), orElseThrow(), map(), flatMap(), filter(). Prefer returning Optional from methods instead of null.

=== [35] ===
PROMPT: What is a Spliterator and how does it relate to parallel streams?
ANSWER: Spliterator is the parallel counterpart of Iterator; it can split its elements for parallel processing. The ForkJoinPool uses Spliterators to divide work. Custom Spliterators can report characteristics (SIZED, SORTED, DISTINCT) that allow the framework to optimize processing.

=== [36] ===
PROMPT: What is the difference between Iterator and ListIterator?
ANSWER: Iterator traverses forward and supports remove(). ListIterator additionally supports backward traversal (hasPrevious/previous), add(), set(), and nextIndex()/previousIndex(). ListIterator is only available for Lists.

=== [37] ===
PROMPT: What are CopyOnWriteArrayList and CopyOnWriteArraySet?
ANSWER: Thread-safe collections where mutating operations (add, set, remove) create a new copy of the underlying array. Iterators never throw ConcurrentModificationException and reflect the snapshot at creation time. Best when reads greatly outnumber writes; unsuitable for write-heavy workloads.

=== [38] ===
PROMPT: What is the difference between Collections.unmodifiableList() and List.of()?
ANSWER: unmodifiableList wraps a mutable list – the view is unmodifiable but the backing list can still be mutated and the view will reflect those changes. List.of() (Java 9+) creates a truly immutable list; mutations throw UnsupportedOperationException and the list cannot be changed through any reference.

=== [39] ===
PROMPT: What is a WeakHashMap and when would you use it?
ANSWER: WeakHashMap holds weak references to keys; entries are automatically removed when the key is garbage collected. Useful for caches where entries should not prevent GC of their keys (e.g. metadata associated with objects you don't own). Not thread-safe; ConcurrentHashMap is preferred for concurrent caches.

=== [40] ===
PROMPT: Explain reduce() in Java Streams with an example.
ANSWER: reduce() combines elements into a single value using an associative function. Example: int sum = list.stream().reduce(0, Integer::sum); The identity element (0) is the starting value. A variant without identity returns Optional<T>. For parallel streams the operation must be associative and stateless.

=== [41] ===
PROMPT: What is the JVM and what are its main components?
ANSWER: The JVM (Java Virtual Machine) executes Java bytecode. Main components: ClassLoader (loads .class files), Runtime Data Areas (Heap, Stack, Method Area, PC Register, Native Method Stack), Execution Engine (Interpreter, JIT Compiler, GC), and Native Method Interface (JNI).

=== [42] ===
PROMPT: What is the difference between the heap and the stack in Java?
ANSWER: Stack: stores method frames (local variables, operand stack, references), one per thread, LIFO, limited size, fast. Heap: shared across threads, stores all object instances and arrays, managed by GC, larger. Stack overflow causes StackOverflowError; heap exhaustion causes OutOfMemoryError.

=== [43] ===
PROMPT: What is the JIT compiler?
ANSWER: The Just-In-Time compiler translates frequently executed bytecode ('hot spots') into native machine code at runtime. HotSpot uses two JITs: C1 (client, fast compilation, less optimised) and C2 (server, slower, heavily optimised). JIT applies inlining, loop unrolling, escape analysis, and other optimisations.

=== [44] ===
PROMPT: What is garbage collection in Java and how does it work?
ANSWER: GC automatically reclaims memory occupied by unreachable objects. Java uses generational GC: Young Generation (Eden + two Survivor spaces) for short-lived objects, Old Generation (Tenured) for long-lived objects. Minor GC collects young gen; Major/Full GC collects old gen. Objects surviving multiple minor GCs are promoted.

=== [45] ===
PROMPT: What are the common GC algorithms in the JVM?
ANSWER: Serial GC – single thread, stop-the-world.
Parallel GC – multi-threaded young gen, still STW, throughput-focused.
G1 GC – region-based, concurrent marking, default since JDK 9, balances throughput and latency.
ZGC – low latency (<10ms pauses), concurrent, scales to terabyte heaps.
Shenandoah – similar to ZGC, RedHat.

=== [46] ===
PROMPT: What is a memory leak in Java and how can it occur?
ANSWER: A memory leak occurs when objects are no longer needed but still reachable (not collected). Common causes: static collections holding references, listeners not removed, caches without eviction, inner classes holding outer references, unbalanced connection/stream management.

=== [47] ===
PROMPT: What is the difference between -Xms and -Xmx JVM flags?
ANSWER: -Xms sets the initial (minimum) heap size; -Xmx sets the maximum heap size. Setting them equal avoids costly heap resizing at startup. Other useful flags: -Xss (thread stack size), -XX:MetaspaceSize/-XX:MaxMetaspaceSize (Metaspace), -XX:+PrintGCDetails.

=== [48] ===
PROMPT: What is Metaspace and how does it differ from PermGen?
ANSWER: PermGen (Java ≤7) stored class metadata in the heap; fixed size, prone to OutOfMemoryError: PermGen. Metaspace (Java 8+) stores class metadata in native memory; grows dynamically (bounded by -XX:MaxMetaspaceSize). This eliminates most PermGen-related OOM errors.

=== [49] ===
PROMPT: What is escape analysis in the JVM?
ANSWER: Escape analysis determines whether an object's reference escapes the current method or thread. If it doesn't escape, the JIT can allocate it on the stack (stack allocation) or eliminate it entirely (scalar replacement), reducing heap pressure and GC pauses.

=== [50] ===
PROMPT: Explain the class loading mechanism in Java.
ANSWER: The ClassLoader subsystem loads, links, and initialises classes. Bootstrap ClassLoader loads rt.jar/JDK classes. Extension/Platform ClassLoader loads extension libs. Application ClassLoader loads classpath classes. Delegation model: a loader delegates to its parent before loading itself (parent-first). Custom ClassLoaders enable hot-reload and isolation.

=== [51] ===
PROMPT: What is the difference between String, StringBuilder, and StringBuffer?
ANSWER: String is immutable; every 'modification' creates a new object. StringBuilder is mutable and not thread-safe; best for single-threaded string building. StringBuffer is mutable and thread-safe (synchronised methods) but slower. The compiler optimises String concatenation with + using StringBuilder internally.

=== [52] ===
PROMPT: What is String interning?
ANSWER: String.intern() returns a canonical representation from the string pool. String literals are automatically interned. Interning saves memory when many identical strings exist, and == can be used for equality comparison. Modern JVMs use G1 string deduplication for heap strings.

=== [53] ===
PROMPT: What is a soft reference, weak reference, and phantom reference?
ANSWER: SoftReference – collected when JVM needs memory (good for memory-sensitive caches). WeakReference – collected eagerly at next GC regardless of memory (WeakHashMap). PhantomReference – collected after finalization; used with ReferenceQueue to take action after an object is collected without using finalize().

=== [54] ===
PROMPT: What causes a StackOverflowError and how to prevent it?
ANSWER: StackOverflowError occurs when the call stack exceeds its limit, typically due to unbounded/infinite recursion. Prevention: ensure base case in recursive methods, convert deep recursion to iteration, increase stack size (-Xss) as a workaround. Tail-call optimisation is not performed by standard JVMs.

=== [55] ===
PROMPT: How does the JVM handle object finalization?
ANSWER: If an object overrides finalize(), the GC may call it before reclaiming the object, but timing is not guaranteed. finalize() is deprecated in Java 9+ due to unpredictability and performance issues. Prefer try-with-resources and Cleaner (Java 9) for resource cleanup.

=== [56] ===
PROMPT: What is the difference between a process and a thread?
ANSWER: A process has its own memory space (heap, code, data, stack). Threads within a process share the heap and code but each has its own stack and program counter. Thread creation is cheaper than process creation. Inter-process communication requires IPC mechanisms; threads share memory directly (with synchronisation needs).

=== [57] ===
PROMPT: What is a race condition and how do you prevent it?
ANSWER: A race condition occurs when multiple threads access shared mutable state concurrently and the result depends on scheduling order. Prevention: synchronisation (synchronized, Lock), atomic variables (AtomicInteger), immutable objects, thread-local storage, or confining mutable state to a single thread.

=== [58] ===
PROMPT: What does the 'synchronized' keyword do in Java?
ANSWER: synchronized on a method or block acquires the intrinsic lock (monitor) of an object. Only one thread can hold the lock at a time; others block. It provides mutual exclusion and visibility (happens-before). Syncing on 'this' for public methods can lead to deadlocks if external code syncs on the same object.

=== [59] ===
PROMPT: What is the 'volatile' keyword in Java?
ANSWER: volatile guarantees visibility: reads/writes to a volatile variable go directly to main memory; cached copies in CPU registers are flushed. It establishes a happens-before relationship. Volatile is not sufficient for compound actions (check-then-act, read-modify-write) – use atomic variables or synchronisation for those.

=== [60] ===
PROMPT: What is a deadlock and how do you detect and prevent it?
ANSWER: Deadlock: two or more threads hold locks and wait for each other's locks, forming a cycle. Detection: thread dump (jstack). Prevention: always acquire locks in consistent global order, use tryLock with timeout (ReentrantLock), prefer lock-free/non-blocking data structures, reduce lock scope.

=== [61] ===
PROMPT: What is the Java Memory Model (JMM)?
ANSWER: The JMM defines how threads interact through memory: what values a thread can see when reading a variable written by another thread. It defines happens-before rules: program order, monitor lock/unlock, volatile write/read, thread start/join, and final field initialisation.

=== [62] ===
PROMPT: What is the difference between ReentrantLock and synchronized?
ANSWER: ReentrantLock (java.util.concurrent.locks): can be acquired without blocking (tryLock), with timeout, interruptibly, and supports fairness. It must be explicitly unlocked in finally. synchronized is simpler, automatically released, and sufficient for most use cases. Both are reentrant.

=== [63] ===
PROMPT: What is the Executor framework and why is it preferred over raw threads?
ANSWER: Executor/ExecutorService decouples task submission from execution. Benefits: thread pooling (avoids creation overhead), bounded queues, task lifecycle management (submit, cancel, await), and richer abstractions (ScheduledExecutorService, ForkJoinPool). Use Executors factory methods or ThreadPoolExecutor directly.

=== [64] ===
PROMPT: What is the difference between Callable and Runnable?
ANSWER: Runnable.run() returns void and cannot throw checked exceptions. Callable<V>.call() returns a value and can throw checked exceptions. Callable is used with ExecutorService.submit() which returns a Future<V> allowing retrieval of the result and exception handling.

=== [65] ===
PROMPT: What is a Future and CompletableFuture?
ANSWER: Future<T> represents an asynchronous computation result; get() blocks until done. CompletableFuture<T> (Java 8) extends Future with non-blocking callbacks (thenApply, thenCompose, thenCombine, exceptionally), composable pipelines, and explicit completion. It integrates with ForkJoinPool.commonPool() by default.

=== [66] ===
PROMPT: Explain the Fork/Join framework.
ANSWER: ForkJoinPool is a work-stealing thread pool optimised for recursive divide-and-conquer tasks. RecursiveTask<V> returns a result; RecursiveAction doesn't. Tasks call fork() to submit subtasks and join() to retrieve results. The common pool is used by parallel streams and CompletableFuture.

=== [67] ===
PROMPT: What is a ThreadLocal variable?
ANSWER: ThreadLocal provides each thread its own independent copy of a variable; threads cannot see each other's copies. Used for per-thread context (user sessions, transactions, SimpleDateFormat instances). Must be cleaned up with remove() in thread pools to prevent memory leaks and data leakage.

=== [68] ===
PROMPT: What are the java.util.concurrent atomic classes?
ANSWER: AtomicInteger, AtomicLong, AtomicBoolean, AtomicReference provide lock-free, thread-safe operations using CAS (Compare-And-Swap). LongAdder/DoubleAdder use striped counters for high-contention counting with better throughput than AtomicLong. LongAccumulator generalises accumulation.

=== [69] ===
PROMPT: What is the difference between CountDownLatch, CyclicBarrier, and Semaphore?
ANSWER: CountDownLatch – one-time gate; threads wait for count to reach 0 (count-down by other threads). CyclicBarrier – reusable barrier; N threads wait for each other to reach a point. Semaphore – controls concurrent access to a resource with a permit count; acquire/release permits.

=== [70] ===
PROMPT: What is the producer-consumer pattern and how to implement it in Java?
ANSWER: Producer generates data and puts it in a shared queue; consumer takes from the queue. Java implementation: BlockingQueue (ArrayBlockingQueue, LinkedBlockingQueue) naturally handles synchronisation with blocking put()/take(). Producers block when full, consumers block when empty.

=== [71] ===
PROMPT: What is a happens-before relationship?
ANSWER: A happens-before (HB) relationship guarantees that all actions (writes) before the HB edge are visible to actions after it. HB rules include: program order within a thread, monitor unlock HB monitor lock, volatile write HB volatile read, thread.start() HB first action in the started thread.

=== [72] ===
PROMPT: What are BlockingQueue implementations and their differences?
ANSWER: ArrayBlockingQueue – bounded, backed by array, fair ordering optional.
LinkedBlockingQueue – optionally bounded, higher throughput.
PriorityBlockingQueue – unbounded, orders by priority.
SynchronousQueue – no capacity, each put blocks until a take.
DelayQueue – unbounded, elements are held until their delay expires.

=== [73] ===
PROMPT: What is livelock and starvation?
ANSWER: Livelock: threads are active but make no progress (e.g. two threads repeatedly yield to each other). Starvation: a thread never gets CPU time because other threads always get priority. Both are detected with thread dumps. Solutions: randomised backoff, fair locks (ReentrantLock(true)).

=== [74] ===
PROMPT: What is ConcurrentHashMap and how does it differ from Hashtable?
ANSWER: ConcurrentHashMap (Java 8+) uses fine-grained synchronisation (CAS + synchronised per-bucket operations); reads are mostly lock-free. Hashtable synchronises every method on the same monitor – poor concurrency. ConcurrentHashMap does not allow null keys or values; allows concurrent reads and a configurable level of write concurrency.

=== [75] ===
PROMPT: Explain the concept of thread safety.
ANSWER: A class/method is thread-safe if it behaves correctly when accessed from multiple threads simultaneously without external synchronisation. Techniques: immutability, confinement (stack or single-thread), synchronisation, lock-free algorithms, concurrent collections, atomic variables.

=== [76] ===
PROMPT: What is the difference between wait() and sleep()?
ANSWER: Thread.sleep(ms) pauses the current thread for a period; it does not release any monitor lock. Object.wait() suspends the current thread and releases the monitor lock, allowing another thread to acquire it. wait() must be called within a synchronized block; it is woken by notify()/notifyAll() or spuriously.

=== [77] ===
PROMPT: What is a StampedLock?
ANSWER: StampedLock (Java 8) supports three modes: exclusive write lock, non-exclusive read lock (like ReentrantReadWriteLock), and optimistic read (no lock acquired – validate stamp after reading). Optimistic reads are very fast and work well for short read-intensive operations with low write contention.

=== [78] ===
PROMPT: What is Dependency Injection in Spring?
ANSWER: DI is a design pattern where an object's dependencies are provided (injected) externally rather than created inside the object. Spring IoC container manages object lifecycle and injects collaborators via constructor injection (recommended), setter injection, or field injection (@Autowired).

=== [79] ===
PROMPT: What is the Spring IoC container?
ANSWER: The IoC (Inversion of Control) container is responsible for creating, configuring, and managing Spring beans (BeanFactory). ApplicationContext extends BeanFactory with AOP, event publishing, i18n, and eager initialisation. Beans are configured via annotations (@Component, @Bean) or XML.

=== [80] ===
PROMPT: What is the difference between @Component, @Service, @Repository, and @Controller?
ANSWER: All are specialisations of @Component enabling component scanning. @Service marks business logic. @Repository marks data access – Spring adds exception translation (DataAccessException wrapping). @Controller marks MVC controllers. Semantically meaningful, functionally equivalent for DI purposes.

=== [81] ===
PROMPT: What is @Autowired and what are the injection types in Spring?
ANSWER: @Autowired injects dependencies by type. Types: constructor (single constructor, no annotation needed in Spring 4.3+, preferred), setter, field (simplest but harder to test). Constructor injection makes dependencies explicit and supports immutable fields. Spring throws NoSuchBeanDefinitionException if no match.

=== [82] ===
PROMPT: What is the difference between @Bean and @Component?
ANSWER: @Component is a class-level annotation for auto-detection via component scanning. @Bean is a method-level annotation inside @Configuration classes used for explicit bean creation – necessary when instantiating third-party classes you cannot annotate.

=== [83] ===
PROMPT: What are Spring bean scopes?
ANSWER: singleton (default) – one instance per container. prototype – new instance per request. request – per HTTP request (web). session – per HTTP session (web). application – per ServletContext (web). websocket – per WebSocket session. Custom scopes are possible.

=== [84] ===
PROMPT: What is @Transactional in Spring and how does it work?
ANSWER: @Transactional wraps a method/class in a database transaction managed by Spring's PlatformTransactionManager. Spring uses AOP proxies: when a bean method is called externally, the proxy begins/commits/rolls back the transaction. Self-invocation bypasses the proxy and the transaction. Propagation and isolation are configurable.

=== [85] ===
PROMPT: What is Spring AOP and what are the key concepts?
ANSWER: AOP (Aspect-Oriented Programming) modularises cross-cutting concerns (logging, security, transactions). Concepts: Aspect (module), JoinPoint (method execution), Pointcut (expression matching joinpoints), Advice (before, after, around), Weaving (applying aspects). Spring AOP uses JDK dynamic proxies or CGLIB.

=== [86] ===
PROMPT: What is the difference between @PathVariable, @RequestParam, and @RequestBody?
ANSWER: @PathVariable extracts a URI template variable (/users/{id}). @RequestParam extracts a query string or form parameter (?name=foo). @RequestBody deserializes the HTTP request body (JSON/XML) into a Java object using Jackson/JAXB. @RequestHeader extracts HTTP headers.

=== [87] ===
PROMPT: What is Spring Boot auto-configuration?
ANSWER: Auto-configuration (@EnableAutoConfiguration / spring.factories or META-INF/spring/auto-configuration-imports) conditionally configures beans based on classpath contents and properties. Example: if H2 and spring-boot-starter-data-jpa are on classpath, a DataSource, EntityManagerFactory, and TransactionManager are auto-configured.

=== [88] ===
PROMPT: What is the difference between @RestController and @Controller?
ANSWER: @Controller is used with view-based MVC; handler methods return view names resolved by ViewResolver. @RestController = @Controller + @ResponseBody; every handler method serialises the return value directly to the HTTP response body (JSON/XML), bypassing view resolution.

=== [89] ===
PROMPT: What is Spring profiles and how do you use them?
ANSWER: Profiles allow different beans/configurations for different environments. Annotate beans/configs with @Profile("dev"). Activate via spring.profiles.active property, SPRING_PROFILES_ACTIVE env var, or programmatically. application-{profile}.properties files are loaded automatically for the active profile.

=== [90] ===
PROMPT: What is @ConditionalOnProperty and when would you use it?
ANSWER: @ConditionalOnProperty registers a bean only when a specified property has a certain value (or exists). Example: @ConditionalOnProperty(name="feature.enabled", havingValue="true") – used for feature flags, optional integrations, or environment-specific components.

=== [91] ===
PROMPT: What is Spring's ApplicationEvent and ApplicationListener?
ANSWER: Spring provides an event mechanism: publish events with ApplicationEventPublisher.publishEvent(), handle with @EventListener annotated methods or ApplicationListener implementations. Built-in events: ContextRefreshedEvent, ContextStartedEvent. Custom events extend ApplicationEvent. Async handling with @Async @EventListener.

=== [92] ===
PROMPT: What is Spring's BeanPostProcessor?
ANSWER: BeanPostProcessor allows custom modification of new bean instances after initialisation. postProcessBeforeInitialization() called before init methods; postProcessAfterInitialization() called after. @Autowired, @PostConstruct processing is done by built-in BeanPostProcessors. Used for proxies, instrumentation.

=== [93] ===
PROMPT: What is Spring's @Value annotation?
ANSWER: @Value injects a property value from application.properties/environment: @Value("${server.port}"). Supports SpEL: @Value("#{systemProperties['user.home']}"). Can provide defaults: @Value("${timeout:30}"). @ConfigurationProperties is preferred for structured configuration binding.

=== [94] ===
PROMPT: Explain Spring's transaction propagation types.
ANSWER: REQUIRED (default) – join existing or create new. REQUIRES_NEW – always create new, suspend existing. MANDATORY – must exist, else exception. SUPPORTS – join if exists, none otherwise. NOT_SUPPORTED – suspend existing. NEVER – must not exist. NESTED – save-point within existing transaction.

=== [95] ===
PROMPT: What is @SpringBootTest and how does it differ from @WebMvcTest?
ANSWER: @SpringBootTest loads the full application context for integration tests. @WebMvcTest loads only the web layer (controllers, filters, converters) with a MockMvc; service/repository beans are mocked with @MockBean. @WebMvcTest is faster; @SpringBootTest is more comprehensive.

=== [96] ===
PROMPT: What is Spring's @ExceptionHandler and @ControllerAdvice?
ANSWER: @ExceptionHandler on a controller method handles exceptions thrown by that controller. @ControllerAdvice (@RestControllerAdvice for REST) is a global exception handler applied across all controllers. Useful for returning consistent error responses (ProblemDetail, RFC 9457).

=== [97] ===
PROMPT: What is Spring Actuator?
ANSWER: Spring Boot Actuator exposes management endpoints (/actuator/health, /actuator/metrics, /actuator/info, /actuator/env, etc.) over HTTP or JMX. Integrates with Micrometer for metrics export to Prometheus, Datadog, etc. Configure exposure with management.endpoints.web.exposure.include.

=== [98] ===
PROMPT: What is the difference between JPA and Hibernate?
ANSWER: JPA (Jakarta Persistence API) is a specification defining ORM annotations and APIs. Hibernate is the most popular implementation. You code to the JPA API; Hibernate provides the runtime. Other implementations: EclipseLink, OpenJPA. Spring Data JPA adds repository abstractions on top.

=== [99] ===
PROMPT: What is an Entity in JPA and what annotations are required?
ANSWER: An entity is a POJO mapped to a database table. Required: @Entity on the class, @Id on the primary key field. Optional: @Table(name=...), @Column(name=...), @GeneratedValue for auto-generated IDs. The entity must have a no-arg constructor (public or protected).

=== [100] ===
PROMPT: What are the JPA entity lifecycle states?
ANSWER: Transient – not associated with a persistence context (new object). Managed – associated with a persistence context; changes are tracked. Detached – was managed, no longer associated. Removed – scheduled for deletion. Transitions: persist (transient→managed), merge (detached→managed), remove (managed→removed), detach.

=== [101] ===
PROMPT: What is the difference between fetch types EAGER and LAZY in JPA?
ANSWER: EAGER – associated entity/collection is loaded immediately with the owning entity (single SQL or join). LAZY – loaded on first access (proxy/placeholder). @ManyToOne and @OneToOne default to EAGER; @OneToMany and @ManyToMany default to LAZY. LAZY is usually preferred for performance; be aware of LazyInitializationException outside a session.

=== [102] ===
PROMPT: What is the N+1 query problem in JPA?
ANSWER: Occurs with LAZY collections: loading N parent entities and accessing their collection triggers N additional queries. Solutions: JOIN FETCH in JPQL, @EntityGraph, batch fetching (@BatchSize), or DTO projections. Use slow-query logs and Hibernate statistics to detect it.

=== [103] ===
PROMPT: What is a JPQL query and how does it differ from SQL?
ANSWER: JPQL (Jakarta Persistence Query Language) queries operate on entity object model (classes and fields) rather than tables and columns. It is database-agnostic; Hibernate translates to the appropriate SQL dialect. Criteria API provides a type-safe programmatic alternative to JPQL strings.

=== [104] ===
PROMPT: What are Spring Data JPA derived query methods?
ANSWER: Spring Data JPA generates queries from method names: findByLastNameAndFirstName, findByAgeGreaterThan, findByEmailContainingIgnoreCase. Supports: And/Or, Is/Equals, Between, LessThan/GreaterThan, Like/Containing, OrderBy, Top/First N. For complex queries use @Query with JPQL or native SQL.

=== [105] ===
PROMPT: What is the @OneToMany / @ManyToOne mapping and which side owns the relationship?
ANSWER: @ManyToOne side owns the relationship (has the foreign key column). @OneToMany side is the inverse (mapped by the @ManyToOne field). Use mappedBy on the @OneToMany to avoid a join table. CascadeType.ALL on @OneToMany is convenient but be careful with CascadeType.REMOVE.

=== [106] ===
PROMPT: What is optimistic locking in JPA?
ANSWER: Optimistic locking uses a @Version field (Integer/Long/Timestamp) on the entity. On update, Hibernate checks the version matches; if another transaction updated it first, an OptimisticLockException is thrown. No actual DB lock is held, so it scales well for low-contention scenarios.

=== [107] ===
PROMPT: What is Flyway and why use it?
ANSWER: Flyway is a database migration tool. Migrations are versioned SQL (or Java) scripts (V1__description.sql). Flyway applies pending migrations in order and records applied versions in flyway_schema_history. Benefits: reproducible schema, version control integration, no manual schema management.

=== [108] ===
PROMPT: What is the difference between a primary key and a foreign key?
ANSWER: Primary key (PK) uniquely identifies each row in a table; must be unique and not null; there is exactly one PK per table. Foreign key (FK) references the PK of another table to enforce referential integrity; can be null (optional relationship); one table can have multiple FKs.

=== [109] ===
PROMPT: What is the difference between INNER JOIN, LEFT JOIN, RIGHT JOIN, and FULL OUTER JOIN?
ANSWER: INNER JOIN – rows with matching values in both tables.
LEFT JOIN – all rows from left + matching from right (NULLs for no match).
RIGHT JOIN – all rows from right + matching from left.
FULL OUTER JOIN – all rows from both, NULLs where no match.

=== [110] ===
PROMPT: What is a database index and how does it improve performance?
ANSWER: An index is a data structure (usually B-tree) that speeds up read queries by allowing the DB to find rows without scanning the full table. Drawback: slows INSERT/UPDATE/DELETE and uses storage. Create indexes on columns used in WHERE, JOIN, ORDER BY. Avoid over-indexing.

=== [111] ===
PROMPT: What is a SQL transaction and what are ACID properties?
ANSWER: A transaction is a unit of work that must complete entirely or not at all. ACID: Atomicity (all or nothing), Consistency (DB moves from one valid state to another), Isolation (concurrent transactions appear sequential), Durability (committed changes survive crashes).

=== [112] ===
PROMPT: What are SQL isolation levels?
ANSWER: READ UNCOMMITTED – dirty reads possible. READ COMMITTED – no dirty reads, non-repeatable reads possible. REPEATABLE READ – no dirty or non-repeatable reads, phantom reads possible (MySQL default). SERIALIZABLE – fully isolated, highest consistency but lowest concurrency. Higher isolation = more locking.

=== [113] ===
PROMPT: What is the difference between WHERE and HAVING?
ANSWER: WHERE filters rows before grouping (operates on individual rows). HAVING filters groups after GROUP BY (operates on aggregated results). You cannot use aggregate functions (SUM, COUNT) in WHERE; use HAVING for that.

=== [114] ===
PROMPT: What is a SQL subquery and when should you use a JOIN instead?
ANSWER: A subquery is a query nested inside another. Correlated subqueries reference the outer query and run for each row (can be slow). JOINs are usually more efficient and readable for combining datasets. Use subqueries for EXISTS checks, derived tables, or when a JOIN would produce duplicate rows.

=== [115] ===
PROMPT: What is database normalisation and what are the first three normal forms?
ANSWER: Normalisation reduces redundancy. 1NF: atomic values, no repeating groups. 2NF: 1NF + no partial dependency on composite PK (non-key attributes depend on the whole PK). 3NF: 2NF + no transitive dependency (non-key attributes depend only on the PK). BCNF extends 3NF for all candidate keys.

=== [116] ===
PROMPT: What is the difference between DELETE, TRUNCATE, and DROP?
ANSWER: DELETE removes specific rows (WHERE clause supported), fires triggers, logged per-row, slower, can be rolled back. TRUNCATE removes all rows quickly (minimal logging), no triggers, cannot be rolled back (DDL in most DBs), resets auto-increment. DROP removes the table and its structure entirely.

=== [117] ===
PROMPT: What is a CTE (Common Table Expression)?
ANSWER: A CTE (WITH clause) defines a temporary named result set used within a SELECT, INSERT, UPDATE, or DELETE. It improves readability and allows recursive queries (WITH RECURSIVE for hierarchical data). CTEs are not materialised by default in most engines (inlined like views).

=== [118] ===
PROMPT: What are window functions in SQL?
ANSWER: Window functions compute values across a set of table rows related to the current row (a 'window') without collapsing rows (unlike GROUP BY). Syntax: FUNCTION() OVER (PARTITION BY col ORDER BY col ROWS BETWEEN ...). Examples: ROW_NUMBER(), RANK(), DENSE_RANK(), LAG(), LEAD(), SUM() OVER.

=== [119] ===
PROMPT: What is the difference between UNION and UNION ALL?
ANSWER: UNION returns distinct rows from both result sets (performs deduplication, slower). UNION ALL returns all rows including duplicates (no dedup, faster). Both require the same number of columns and compatible types. Use UNION ALL when you know there are no duplicates or don't care.

=== [120] ===
PROMPT: What is a database view and when would you use one?
ANSWER: A view is a named query stored in the DB. It simplifies complex queries, provides a security layer (exposing only certain columns/rows), and can centralise business logic. Updatable views allow DML in some DBs. Materialised views store query results physically for performance.

=== [121] ===
PROMPT: What is the difference between optimistic and pessimistic locking in databases?
ANSWER: Pessimistic locking acquires DB locks (SELECT FOR UPDATE) upfront, preventing concurrent modifications; suitable for high-contention scenarios but limits scalability. Optimistic locking uses a version check at commit time (no DB lock held); suitable for low-contention, read-heavy scenarios.

=== [122] ===
PROMPT: What is an execution plan and how do you use it?
ANSWER: An execution plan (EXPLAIN / EXPLAIN ANALYZE in PostgreSQL) shows how the query optimiser executes a query: join strategy (nested loop, hash join), index usage, row estimates, and costs. Identify table scans, expensive sorts, and missing indexes. Use it to tune slow queries.

=== [123] ===
PROMPT: What is the difference between clustered and non-clustered indexes?
ANSWER: Clustered index defines the physical order of rows in a table (InnoDB PK is clustered). There can be only one per table. Non-clustered (secondary) indexes store the index key plus a pointer to the actual row (PK value in InnoDB). Multiple non-clustered indexes are allowed.

=== [124] ===
PROMPT: What is sharding in databases?
ANSWER: Sharding is horizontal partitioning: splitting a large table across multiple database instances (shards) based on a shard key (e.g. user ID mod N). Benefits: scales writes and storage. Drawbacks: cross-shard queries are complex, re-sharding is expensive, joins across shards are difficult.

=== [125] ===
PROMPT: What is connection pooling?
ANSWER: Connection pooling maintains a cache of database connections that can be reused, avoiding the overhead of creating/closing a new connection per request. HikariCP (Spring Boot default) is a high-performance pool. Configure minimumIdle, maximumPoolSize, connectionTimeout, idleTimeout.

=== [126] ===
PROMPT: What are stored procedures and their trade-offs?
ANSWER: Stored procedures are precompiled SQL routines stored in the DB. Benefits: reduced network round-trips, encapsulated business logic, reusability. Drawbacks: harder to version control, tightly coupled to DB engine, difficult to test, can hide business logic from application developers.

=== [127] ===
PROMPT: What is the CAP theorem?
ANSWER: CAP theorem states a distributed system can guarantee at most two of: Consistency (every read sees the latest write), Availability (every request gets a response), Partition tolerance (system works despite network partitions). Since partitions are inevitable, real systems choose CP (consistent, may be unavailable) or AP (available, may be stale).

=== [128] ===
PROMPT: What is the difference between horizontal and vertical scaling?
ANSWER: Vertical scaling (scale up): add more CPU/RAM/disk to existing servers; simpler, limited by hardware ceiling. Horizontal scaling (scale out): add more servers; requires stateless services, load balancing, distributed coordination. Cloud-native architectures favour horizontal scaling.

=== [129] ===
PROMPT: What is a load balancer and what algorithms does it use?
ANSWER: A load balancer distributes incoming requests across server instances. Algorithms: Round Robin, Least Connections, IP Hash (session affinity), Weighted RR. Layer 4 LB routes by TCP/IP; Layer 7 LB routes by content (URL, headers). Examples: NGINX, HAProxy, AWS ALB.

=== [130] ===
PROMPT: What is a microservices architecture?
ANSWER: An architectural style where an application is composed of small, independently deployable services, each owning its data and communicating via APIs (REST, gRPC) or message queues. Benefits: independent scaling/deployment, technology flexibility. Drawbacks: distributed system complexity, network latency, data consistency challenges.

=== [131] ===
PROMPT: What is an API Gateway?
ANSWER: An API Gateway is a reverse proxy that is the single entry point for clients. It handles cross-cutting concerns: routing, authentication, rate limiting, SSL termination, request aggregation, and observability. Examples: Kong, AWS API Gateway, Spring Cloud Gateway.

=== [132] ===
PROMPT: What is event-driven architecture and what are its benefits?
ANSWER: Services communicate via events (messages) through a broker (Kafka, RabbitMQ). Producer publishes events without knowing consumers (loose coupling). Benefits: decoupling, scalability, resilience, audit trail. Drawbacks: eventual consistency, harder debugging, message ordering complexity.

=== [133] ===
PROMPT: What is a circuit breaker pattern?
ANSWER: A circuit breaker prevents cascading failures: when a downstream service fails repeatedly, the circuit 'opens' and fails fast (returns error or fallback) without calling the unhealthy service. After a timeout it 'half-opens' to test recovery. Implementations: Resilience4j, Hystrix (deprecated).

=== [134] ===
PROMPT: What is CQRS?
ANSWER: Command Query Responsibility Segregation separates read (query) and write (command) models. Write side handles state changes; read side is optimised for queries (can use denormalised read models). Often combined with Event Sourcing. Complexity trade-off: useful for high-throughput, complex domains.

=== [135] ===
PROMPT: What is Event Sourcing?
ANSWER: Instead of storing current state, Event Sourcing stores a sequence of events (what happened). Current state is rebuilt by replaying events. Benefits: full audit trail, temporal queries, event replay. Drawbacks: eventual consistency, complex queries, event schema evolution.

=== [136] ===
PROMPT: What is a cache and what caching strategies exist?
ANSWER: A cache stores frequently accessed data for fast retrieval. Strategies: Cache-Aside (app manages cache), Read-Through (cache fetches on miss), Write-Through (write to cache and DB simultaneously), Write-Behind (write to cache, async to DB). Eviction policies: LRU, LFU, TTL.

=== [137] ===
PROMPT: What is rate limiting and how is it implemented?
ANSWER: Rate limiting restricts number of requests per client per time window. Algorithms: Token Bucket (smooth, allows bursts), Leaky Bucket (smooth output), Fixed Window Counter (simple, boundary reset), Sliding Window Log (accurate, memory-intensive). Implemented at API Gateway or in code (Bucket4j, Resilience4j).

=== [138] ===
PROMPT: What is the difference between REST and gRPC?
ANSWER: REST uses HTTP/1.1, JSON, human-readable, widely supported, stateless. gRPC uses HTTP/2, Protocol Buffers (binary, compact, faster), supports streaming (unary, server/client/bidi), auto-generated type-safe clients. gRPC excels for internal microservice communication; REST for public APIs.

=== [139] ===
PROMPT: What is the Saga pattern?
ANSWER: Saga manages distributed transactions across microservices without two-phase commit. Two styles: Choreography (services emit events and react to each other's events) and Orchestration (a central orchestrator commands each service). Compensating transactions roll back on failure.

=== [140] ===
PROMPT: What is the difference between unit, integration, and end-to-end tests?
ANSWER: Unit tests verify a single class/method in isolation (mocks for dependencies), fast, numerous. Integration tests verify the interaction of multiple components (DB, Spring context), slower. E2E tests exercise the full system via UI or API, slowest, fewest. Follow the Testing Pyramid (many units, fewer integration, fewest E2E).

=== [141] ===
PROMPT: What is JUnit 5 and what are its key annotations?
ANSWER: JUnit 5 (Jupiter) is the standard Java testing framework. Key annotations: @Test, @BeforeEach, @AfterEach, @BeforeAll, @AfterAll, @Disabled, @DisplayName, @ParameterizedTest, @ValueSource, @CsvSource, @ExtendWith. @Nested allows hierarchical tests. Assertions are in org.junit.jupiter.api.Assertions.

=== [142] ===
PROMPT: What is Mockito and how do you use it?
ANSWER: Mockito creates mock objects for unit testing. @Mock creates a mock; @InjectMocks injects mocks into the tested class; @Spy wraps a real object. when(mock.method()).thenReturn(value) stubs behaviour. verify(mock).method() checks interactions. @ExtendWith(MockitoExtension.class) initialises annotations.

=== [143] ===
PROMPT: What is the difference between a mock and a stub?
ANSWER: Stub: returns predefined responses; doesn't verify behaviour. Mock: additionally verifies that methods were called with expected arguments. Mockito supports both patterns. Spies are partial mocks (real object with some methods stubbed). Fakes are working implementations used in tests (in-memory DB).

=== [144] ===
PROMPT: What is TDD (Test-Driven Development)?
ANSWER: TDD is a development approach: Red (write a failing test), Green (write minimum code to pass), Refactor (improve code while tests stay green). Benefits: forces clear requirements, better design, higher test coverage, confidence in refactoring. Tests serve as living documentation.

=== [145] ===
PROMPT: What is @SpringBootTest vs @DataJpaTest?
ANSWER: @SpringBootTest starts a full Spring Boot context (all beans, embedded server if RANDOM_PORT). @DataJpaTest loads only JPA-related beans (repositories, DataSource, EntityManager) with an in-memory H2 DB by default; no web layer. @DataJpaTest is faster for testing repository/JPA logic.

=== [146] ===
PROMPT: What is MockMvc and how is it used?
ANSWER: MockMvc tests Spring MVC controllers without starting a real HTTP server. Use @WebMvcTest + @Autowired MockMvc. Perform requests: mockMvc.perform(get("/api/users")).andExpect(status().isOk()).andExpect(jsonPath("$.name").value("Alice")). Use MockMvcRequestBuilders, MockMvcResultMatchers.

=== [147] ===
PROMPT: What is parameterized testing in JUnit 5?
ANSWER: @ParameterizedTest runs a test multiple times with different arguments. Sources: @ValueSource (simple values), @CsvSource (CSV strings), @CsvFileSource (from file), @MethodSource (factory method returning Stream), @EnumSource. Reduces test duplication for data-driven scenarios.

=== [148] ===
PROMPT: What is AssertJ and how does it differ from JUnit assertions?
ANSWER: AssertJ provides a fluent, readable assertion API: assertThat(list).hasSize(3).contains("a").doesNotContain("b"). It gives better error messages and supports rich assertions for collections, strings, exceptions, optionals. Preferred over JUnit's static Assertions for readability.

=== [149] ===
PROMPT: What is test coverage and what are its limitations?
ANSWER: Test coverage measures how much of the codebase is exercised by tests (line, branch, method). Tools: JaCoCo, IntelliJ. Limitations: 100% coverage does not mean zero bugs; tests may pass without meaningful assertions; coverage does not guarantee correct behaviour, only that code was executed.

=== [150] ===
PROMPT: What is a flaky test?
ANSWER: A flaky test produces inconsistent results (pass/fail) without code changes. Causes: timing issues (Thread.sleep), order dependency, shared mutable state, reliance on external services, random data. Fix with deterministic test data, proper async waiting (Awaitility), test isolation, and mocking external services.

=== [151] ===
PROMPT: What is WireMock?

=== [152] ===
PROMPT: What is Testcontainers?

=== [153] ===
PROMPT: What is Maven and what are its key concepts?
ANSWER: Maven is a build and dependency management tool. Key concepts: POM (Project Object Model – pom.xml), artifact (group:artifact:version), repository (local ~/.m2, central, remote), lifecycle (compile, test, package, install, deploy), phases, plugins, and goals.

ANSWER: A BOM is a POM with <dependencyManagement> that defines versions of a set of related dependencies. Consumers import it with <scope>import</scope> and don't need to specify versions. Spring Boot parent POM is a BOM. Ensures consistent versions across multi-module projects.

=== [157] ===
PROMPT: What is Docker and how does it relate to Java microservices?
ANSWER: Docker packages applications with their dependencies into containers (images). For Java: write a Dockerfile (FROM eclipse-temurin:17, COPY jar, ENTRYPOINT), build image, run container. === [159] ===
PROMPT: What is CI/CD?
ANSWER: Continuous Integration: developers merge code frequently; automated build and tests run on each push catching integration bugs early. Continuous Delivery: every passing build is automatically deployable. Continuous Deployment: every passing build is automatically deployed to production. Tools: GitHub Actions, Jenkins, GitLab CI.

=== [160] ===

ANSWER: Gradle uses Groovy/Kotlin DSL (build.gradle), is more flexible and faster (incremental builds, build cache, daemon). Maven uses declarative XML, convention-over-configuration, more standardised. Gradle is preferred for large multi-project builds (Android, Kotlin); Maven is ubiquitous in Java enterprise.

=== [161] ===
PROMPT: What is semantic versioning?
ANSWER: SemVer format: MAJOR.MINOR.PATCH. MAJOR incremented for incompatible API changes. MINOR for backwards-compatible new features. PATCH for backwards-compatible bug fixes. Pre-release: 1.0.0-alpha, 1.0.0-SNAPSHOT. Helps consumers understand upgrade impact.

=== [162] ===
PROMPT: What is a health check in a Spring Boot application?
ANSWER: Spring Boot Actuator's /actuator/health endpoint reports application health. It aggregates indicators: DataSourceHealthIndicator, DiskSpaceHealthIndicator, custom HealthIndicator beans. === [163] ===
PROMPT: What is the difference between @MockBean and @Mock in Spring tests?
ANSWER: @Mock (Mockito) creates a mock managed by Mockito, used in pure unit tests with MockitoExtension. @MockBean (Spring Boot Test) creates a Mockito mock and registers it as a Spring bean in the application context, replacing any existing bean of the same type. === [164] ===
PROMPT: What is the difference between eager and lazy initialization of Spring beans?
ANSWER: Eager (default): all singleton beans are created at ApplicationContext startup. Lazy (@Lazy or spring.main.lazy-initialization=true): beans are created on first access. Lazy init speeds startup but may delay errors to runtime. Useful in large applications or tests where most beans aren't needed.

=== [165] ===
PROMPT: What is a Spring Boot starter?
ANSWER: A starter is a dependency descriptor (POM) that pulls in all necessary dependencies for a feature. spring-boot-starter-web brings Tomcat, Spring MVC, Jackson. spring-boot-starter-data-jpa brings Hibernate, Spring Data JPA, HikariCP. Starters reduce dependency management boilerplate.

=== [166] ===
PROMPT: What is the difference between synchronous and asynchronous programming?
ANSWER: Synchronous: caller blocks until the operation completes. Asynchronous: caller submits work and continues; result is obtained later via callback, Future, or reactive streams. Async improves throughput for I/O-bound tasks by freeing threads while waiting. Complexity: callbacks, error handling, debugging.

=== [167] ===
PROMPT: What is Project Reactor and reactive programming?
ANSWER: Project Reactor is a reactive library (Mono<T> for 0-1 items, Flux<T> for N items) based on Reactive Streams specification. Operators (map, flatMap, filter, zip) compose non-blocking pipelines. Spring WebFlux uses Reactor for async, non-blocking web handling on Netty.

=== [168] ===
PROMPT: What is Spring WebFlux and when would you use it over Spring MVC?
ANSWER: Spring WebFlux is a reactive, non-blocking web framework using Reactor. Use when: handling many concurrent connections with few threads (chat, streaming), I/O-bound operations, or consistency with reactive data stores (R2DBC, Reactive MongoDB). Avoid for CPU-bound, or when team is unfamiliar with reactive.

=== [169] ===
PROMPT: What is the difference between PUT and PATCH in REST?
ANSWER: PUT replaces the entire resource (idempotent; all fields must be sent). PATCH partially updates a resource (semantically not required to be idempotent; only changed fields sent). POST creates a new resource (not idempotent). GET/HEAD/PUT are idempotent by specification.

=== [170] ===
PROMPT: What are HTTP status codes and what do the ranges mean?
ANSWER: 1xx – Informational. 2xx – Success (200 OK, 201 Created, 204 No Content). 3xx – Redirection (301 Moved Permanently, 304 Not Modified). 4xx – Client errors (400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 409 Conflict, 422 Unprocessable Entity). 5xx – Server errors (500, 503).

=== [171] ===
PROMPT: What is JWT authentication?
ANSWER: JWT (JSON Web Token) consists of Header.Payload.Signature (Base64 encoded). Server signs with secret/private key; client stores token and sends in Authorization: Bearer header. Server validates signature without DB lookup (stateless). Claims in payload. Refresh tokens handle expiry. Do not store sensitive data in payload.

=== [172] ===
PROMPT: What is the difference between authentication and authorisation?
ANSWER: Authentication verifies identity (who are you? – username/password, token, biometric). Authorisation determines permissions (what can you do? – roles, ACLs). Spring Security: authentication by AuthenticationManager/AuthenticationProvider; authorisation by AccessDecisionManager, @PreAuthorize, hasRole().

=== [173] ===
PROMPT: What is a deadlock in SQL databases?
ANSWER: A SQL deadlock occurs when two transactions each hold a lock the other needs, forming a circular wait. DBs detect and kill one transaction (deadlock victim) and roll it back. Prevention: acquire locks in consistent order, keep transactions short, use appropriate isolation level.

=== [174] ===
PROMPT: What is the difference between an optimistic and pessimistic read in Hibernate?
ANSWER: Optimistic read: no lock; relies on @Version check at commit to detect concurrent modification. Pessimistic read: SELECT FOR SHARE or SELECT FOR UPDATE; holds a DB row lock for the duration of the transaction. Pessimistic prevents conflicts upfront but reduces throughput.

ANSWER: A composite (multi-column) index covers multiple columns: CREATE INDEX idx ON table(a,b,c). Queries using a leftmost prefix of the index columns benefit. A query on (a,b) benefits; a query on (b) alone typically doesn't. Column order matters: high-cardinality, frequently filtered columns first.

PROMPT: What is lazy loading vs eager loading trade-offs in Spring Data JPA?
ANSWER: Lazy loading avoids loading data you don't need, reducing DB queries and memory for simple use cases. But accessing a lazy field outside an open session throws LazyInitializationException. Eager loading fetches everything upfront, safe but may load unnecessary data and cause performance issues with large collections.

ANSWER: Projections return only a subset of entity data. Interface-based: define an interface with getter methods matching field names; Spring Data generates a proxy. Class-based (DTO): constructor expression in JPQL or @Query with constructor call. Projections avoid loading unnecessary data and mapping overhead.

ANSWER: Pattern matching instanceof: if (obj instanceof String s) { s.length(); } – eliminates explicit cast. Switch expressions (Java 14) and pattern matching switch (Java 21) allow matching on types and destructuring. Enhances readability and reduces boilerplate.

ANSWER: Text blocks are multi-line string literals delimited by triple quotes ("""). They preserve formatting, handle escape sequences, and strip common leading whitespace. Ideal for JSON/SQL/HTML strings in source code without manual newline escaping. Trailing spaces can be preserved with \s.

ANSWER: JPMS (Project Jigsaw) introduces modules: named sets of packages with explicit dependencies (requires) and exports (exports). Defined in module-info.java. PROMPT: What is the var keyword in Java (Java 10)?
ANSWER: var enables local variable type inference: var list = new ArrayList<String>(); The type is inferred by the compiler at compile time (still statically typed). Improves readability for long generic types. Cannot be used for fields, method parameters, return types, or with null/uninitialized variables.

PROMPT: What is method reference and what are the four types?
ANSWER: Method references are shorthand for lambdas that call a single method. Types: static (ClassName::staticMethod), instance method on particular instance (instance::method), instance method on arbitrary instance of type (ClassName::instanceMethod), and constructor (ClassName::new).

PROMPT: What is a lambda expression in Java?
ANSWER: A lambda is an anonymous function: (params) -> expression or (params) -> { statements; }. It implements a functional interface. Lambdas have access to effectively final local variables from enclosing scope. The compiler infers the functional interface from context (parameter type, return type).

=== [192] ===
PROMPT: What is the difference between Comparable.compareTo() returning negative, zero, positive?
ANSWER: compareTo returns negative if this < other, zero if equal, positive if this > other. Consistent with equals() is recommended (though not required). Used by Collections.sort(), TreeMap, TreeSet for natural ordering. Violating this contract (inconsistent with equals) can cause unexpected behaviour in sorted collections.

=== [193] ===
PROMPT: What is the difference between a shallow and deep copy of a collection?
ANSWER: Shallow copy (new ArrayList<>(original)): creates a new collection with the same element references. Mutating a mutable element affects both. Deep copy: recursively copies elements. For deep copy: iterate and clone each element, use serialisation, or a copy constructor pattern.

=== [194] ===
PROMPT: What are the SOLID principles applied to Spring beans?
ANSWER: S: one Spring bean = one responsibility (e.g. separate service from repository). O: extend via composition/decoration, not modification. L: substitutable implementations of the same interface (e.g. DataSource, MessageSender). I: fine-grained interfaces (don't force beans to depend on methods they don't use). D: inject abstractions via @Autowired.

=== [195] ===
PROMPT: What is the difference between pessimistic and optimistic concurrency control in Java?
ANSWER: Pessimistic (synchronized/Lock): assumes conflicts will happen; holds lock, serialises access. Optimistic (AtomicReference CAS, @Version): assumes conflicts are rare; attempts update, retries if conflict detected. Optimistic is better for low-contention read-heavy workloads; pessimistic for high-contention write-heavy scenarios.

=== [196] ===
PROMPT: What is a memory-mapped file in Java?
ANSWER: Memory-mapped files (java.nio.MappedByteBuffer) map a file region into virtual memory, allowing file I/O with memory access semantics. The OS handles paging. Useful for large file processing (logs, binary data) with better performance than stream I/O. Files.map() provides access via FileChannel.

=== [197] ===
PROMPT: What is NIO (Non-blocking I/O) in Java?
ANSWER: java.nio introduces Channels, Buffers, and Selectors. A Selector multiplexes I/O events from multiple channels on a single thread (reactor pattern). This enables high-throughput network servers without one-thread-per-connection overhead. Netty and Java's AsynchronousChannels build on this.

=== [198] ===
PROMPT: What is the difference between checked and unchecked exceptions and best practices for each?
ANSWER: Checked exceptions (IOException, SQLException) force callers to handle them; use for recoverable conditions where the caller can do something meaningful. Unchecked (RuntimeException) for programming errors or truly unrecoverable situations. Modern APIs (including Spring) prefer unchecked; avoid swallowing exceptions silently.

=== [199] ===
PROMPT: What is a service mesh and what problem does it solve?
ANSWER: A service mesh (Istio, Linkerd) provides networking infrastructure for microservices: service discovery, load balancing, mTLS encryption, circuit breaking, retries, and observability without application code changes. A sidecar proxy (Envoy) is injected alongside each service. Solves distributed systems concerns uniformly.

=== [200] ===
PROMPT: What is Kubernetes and its core concepts?
ANSWER: Kubernetes (K8s) is a container orchestration platform. Core concepts: Pod (one or more containers), Deployment (desired state, rolling updates), Service (stable network endpoint), ConfigMap/Secret (config), PersistentVolume, Namespace, Ingress. === [201] ===
PROMPT: What is an index hint and when would you use it?
ANSWER: An index hint (USE INDEX, FORCE INDEX in MySQL; hints in Oracle/PostgreSQL) overrides the query optimiser's index selection. Use when the optimiser consistently chooses a poor plan for a specific query. Prefer as last resort; updating statistics or rewriting the query is usually better.

=== [202] ===
PROMPT: What is the purpose of @Transactional(readOnly=true)?
ANSWER: readOnly=true signals to the transaction manager and Hibernate that no writes will occur. Hibernate can skip dirty checking at flush time (performance), and the DB may use read-only optimisations. Some databases/drivers allow routing read-only transactions to a read replica.

ANSWER: persist() makes a transient entity managed; entity must not already have an ID. merge() returns a new managed instance with state copied from the detached instance; the original detached instance remains detached. Use merge when working with detached entities (e.g. coming from web tier).

=== [204] ===

ANSWER: @EnableScheduling activates scheduled task support. @Scheduled(fixedRate=5000) runs a method every 5 seconds. @Scheduled(cron="0 0 * * * *") uses cron expression. ANSWER: Context refresh loads/creates all bean definitions, runs BeanFactoryPostProcessors (config processing, @Value resolution), instantiates and wires singleton beans, runs BeanPostProcessors (AOP proxies), calls @PostConstruct methods, and publishes ContextRefreshedEvent. @DependsOn controls initialization order.

=== [212] ===
PROMPT: What are varargs in Java?
ANSWER: Varargs (variable-length arguments) allow a method to accept zero or more arguments of a type: void log(String... messages). Internally an array. Must be the last parameter. Beware with generics (heap pollution warning). Call with individual args, an array, or no args.

=== [213] ===
PROMPT: What is the diamond problem in Java and how does Java solve it?
ANSWER: The diamond problem occurs when a class inherits from two interfaces with the same default method. Java 8+ requires the implementing class to explicitly override the conflicting method (otherwise compile error). === [214] ===
PROMPT: What is the difference between fail-fast and fail-safe iterators?
ANSWER: Fail-fast iterators (ArrayList, HashMap) throw ConcurrentModificationException if the collection is modified structurally during iteration (detected via modCount). Fail-safe iterators (CopyOnWriteArrayList, ConcurrentHashMap) iterate over a snapshot or use internal synchronisation; no exception thrown, may see stale data.

=== [215] ===
PROMPT: What is a Phantom Read in SQL and which isolation level prevents it?
ANSWER: A phantom read occurs when a transaction re-executes a range query and sees new rows inserted by another committed transaction. REPEATABLE READ prevents non-repeatable reads (re-reading same row) but not phantoms. SERIALIZABLE prevents phantom reads by locking the range.

=== [216] ===
PROMPT: What are the differences between HashMap and ConcurrentHashMap?
ANSWER: HashMap: not thread-safe; external synchronisation required; allows null key and values. ConcurrentHashMap: thread-safe with fine-grained locking (CAS + segment locks in Java 8+); does not allow null keys or values; compute operations are atomic; higher throughput than Hashtable.

=== [221] ===

ANSWER: Two concurrent requests read the same row, both check a condition (e.g. available stock > 0), both pass and both update – overselling.
